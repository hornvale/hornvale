**Goal:** Derive each species' carrying-capacity field from its traits coupled to the world's environmental fields (`K = resource-supply × Π condition-response`), so that composition varies across space and the merged coexistence packer produces real biogeography instead of a uniform blend.

**Architecture:** An entity-agnostic **suitability engine** in the kernel (a condition-response curve + a substrate-field abstraction), consumed by a new worldgen layer that computes per-species K from `(niche, condition_niche, sovereignty)` × the already-shipped environmental fields. The demography packer (Phase 1, merged) is **unchanged** — only its `per_species_k` input gets richer. Built critical-path first: prove biogeography emerges on the current 4-species roster (A→E) before authoring the menagerie (F).

**Tech Stack:** Rust 2024, `serde`/`serde_json` only. Kernel: `Geosphere`, `CellId`, `CellMap`, `Mass`, `ResourceVector`, `hornvale_kernel::math`. Shipped field accessors: `climate.mean_temperature_at`, `climate.moisture_at`, `terrain.elevation_at`, astronomy insolation. Runner: `cargo nextest`.

## Global Constraints

- **Determinism is constitutional.** Same seed + pins → byte-identical. K draws **nothing** from the seed (pure function of authored traits × world fields). No `HashMap`/`HashSet` (`BTreeMap`/`Vec`); `total_cmp` for float sorts.
- **All `f64` transcendentals route through `hornvale_kernel::math`.** The condition Gaussian introduces `exp` into the compute path → add/route `math::exp` (confirm whether `exp` exists in `kernel/src/math.rs`; if absent, add it there, libm-routed, beside `powf`). Never inherent `f64::exp`. `sqrt` stays inherent (IEEE-754 exact, per `math.rs` doc). Quantize only at emit.
- **The packer does not change.** `domains/demography` still consumes `per_species_k: &[(u32, CellMap<f64>)]`. This campaign only changes how worldgen computes that input.
- **Layering (`cli/tests/architecture.rs`):** kernel → domains → windows → cli. `domains/species` and `domains/demography` depend only on kernel; shared types live in kernel. `windows/worldgen` is the composition root — the only place fields meet species traits.
- **Every crate `#![warn(missing_docs)]`;** every pub item/field/variant documented.
- **Type-audit (default-deny):** every `pub`-boundary primitive carries a ratified `type-audit:` tag (valid classes ONLY — consult `tools/type-audit/src/tag.rs`; never invent one, keep tuple/struct internals private where possible). Verify `cargo run --manifest-path tools/type-audit/Cargo.toml -- check`.
- **Save-format:** the K formula and its frozen constants (Gaussian, saturation, sovereignty mapping, aggregation choice) become a save-format contract when frozen; declare them as named `const`s with CALIBRATED/AUTHORED provenance (mirror `carrying_capacity.rs`). Aggregation = **product-of-Gaussians** (G3 decision); sovereignty **derived from mass+potency with an authored override** (G3).
- **`cargo fmt` is the final step before every commit.** Commit gate = `make gate`. The census stays in the deferred-shared-AWS-regen red batch (do NOT regen locally); re-pin only local artifacts.
- **Conform to the shipped `Domain` trait / registration** where predicates are emitted (this campaign emits none until the cutover).

### Task A2: `ConditionResponse` — the sovereignty-floored, devotion-peaked curve

**Files:** Modify `kernel/src/ecology.rs`; Modify `kernel/src/lib.rs` (re-export).

**Interfaces:**
- Produces: `ConditionResponse { optimum: f64, width: f64, devotion: f64 }` and `pub fn eval(&self, field: f64, floor: f64) -> f64` = `floor + (1.0 - floor) * self.devotion * math::exp(-0.5 * ((field - optimum)/width)^2)`, clamped to `[0,1]` (devotion may push the peak; clamp). `floor ∈ [0,1]` supplied by the caller (from sovereignty). Doc every field.

- [ ] **Step 1:** Failing test in `ecology.rs`:
```rust
#[test]
fn condition_response_floors_and_peaks() {
    let r = ConditionResponse { optimum: 20.0, width: 10.0, devotion: 1.0 };
    // hard (floor 0): excluded far from optimum, ~1 at optimum
    assert!((r.eval(20.0, 0.0) - 1.0).abs() < 1e-9);
    assert!(r.eval(80.0, 0.0) < 0.01, "hard tolerance excludes far away");
    // soft (floor 0.7): never excluded, still peaked
    assert!(r.eval(80.0, 0.7) >= 0.7, "sovereign floor never excludes");
    assert!(r.eval(20.0, 0.7) > r.eval(80.0, 0.7), "still prefers its optimum");
}
```
- [ ] **Step 2:** Run → FAIL. **Step 3:** Implement (route `exp` through `math::exp`; clamp result to `[0,1]`). **Step 4:** PASS. **Step 5:** type-audit (`bare-ok(ratio: ...)`/`bare-ok(diagnostic-value: ...)` per ratified set) + fmt + commit `feat(kernel): ConditionResponse (sovereignty-floored, devotion-peaked)`.
