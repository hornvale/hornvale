# The Alarm — Implementation Plan

**Status: COMPLETE (shipped 2026-07-22).** All three tasks landed (T1 99b0aa10,
T2 c7254c13, T3 0e010c87); seed-42 byte-identical, health null-control green,

Fear-contagion (PSY-11's reserved instance). Spec:
`docs/superpowers/specs/2026-07-21-the-alarm-design.md`. A per-tick **alarm
field** summed over the frozen population's PRIMARY-afraid creatures, read
**additively + latently** into the existing `Danger` drive. Not a new drive —
the externally-driven substitution of the self-driven one. Field-not-agents
(PSY-12 determinism); primary-only emission ⇒ termination for free; derived,
no epoch.

**Goal:** A calm creature standing beside genuine distress catches the alarm and
flees, then settles once separated — a bounded, terminating wave one hop beyond
real hazard ground — while every current world stays byte-identical.

## Global Constraints

- No `HashMap`/`HashSet` — `BTreeMap`/`BTreeSet`/`Vec` only. Float order via
  `f64::total_cmp` with deterministic tie-breaks.
- No wall-clock time (`WorldTime { day: f64 }`). `f64` transcendentals via
  `hornvale_kernel::math`.
- **Byte-identity:** `new --seed 42` and the seed-42 possession galleries are
  byte-for-byte unchanged (the alarm term is dormant — the settled peoples never
  reach primary danger distress, so the field is empty on seed 42). The gate's
  artifact-drift check enforces it.
- **Determinism:** the alarm field is DERIVED from committed facts — no seed
  draw, no new predicate, **no epoch**. The field is a compute-path intermediate,
  never serialized (only the `agent-at` moves it influences commit, and those
  already quantize). The sum is order-independent (`BTreeMap`-keyed).
- Every crate `#![warn(missing_docs)]`; every pub item documented.
- **type-audit:** every primitive at a `pub` boundary carries a `type-audit:`
  verdict tag; run `cargo run --manifest-path tools/type-audit/Cargo.toml --
  check` before each commit.
- Dependencies: `serde` + `serde_json` only. All work lives in `windows/vessel`
  (+ a `windows/lab` scenario in Task 3). No worldgen change (the field is
  dynamic/per-tick, a vessel concern — not a static composition-root field).
- `cargo fmt` is the final step before every commit. Iterate cost-ordered
  (fmt + clippy, then scoped tests); `make gate` is the final pre-commit step
  for boundary changes.
- Test terrains use the new `Default`-based builders (`PlantedTerrain` /
  `SyntheticTerrain`, landed 9f57d7a6) — a new field is one struct-field
  default, not N constructor edits.

### Task 1: The additive-latent `Danger` alarm read

The read side, self-contained and testable by planting an alarm map directly
(no field builder yet). Implements spec §2.

**Files:**
- Modify: `windows/vessel/src/liveness.rs` (the `Danger` struct + `urgency`;
  the `ALARM_SCALE` const)
- Test: `windows/vessel/src/liveness.rs` `#[cfg(test)]` module (unit tests
  beside the existing Danger tests)

**Interfaces:**
- `Danger<'a>` gains a field `alarm: Option<&'a std::collections::BTreeMap<RoomAddr, f64>>`
  — the per-tick alarm field (sparse, RoomAddr-keyed to match the drive's own
  `view.position`/`neighbors()` reads; `None` ⇒ no contagion, the current
  behaviour). Documented; no primitive at the boundary (map of typed keys),
  so no new type-audit tag beyond the existing struct.
- `const ALARM_SCALE: f64` — the latent scale on borrowed alarm (authored;
  pinned in Task 2 against the seed-42 probe, but a sane default here, e.g.
  `1.0`, since seed 42's field is empty regardless).
- `Danger::urgency` folds the alarm at `view.position` into the felt threat
  BEFORE the boldness scaling:
  ```
  base = max over cell+neighbours of threat_at(room)          // unchanged
  felt = base + ALARM_SCALE * alarm_at(view.position)         // NEW, 0.0 if None
  urgency = (felt * mettle_factor()).clamp(0.0, 1.0)
  ```
  Alarm is read at the creature's OWN cell only (the halo already spread it to
  neighbours in the field build — reading neighbours again would double-count).

**Steps (TDD):**

  - `borrowed_alarm_is_scaled_by_boldness`: same planted alarm, a bold creature

### Task 2: The `alarm_field` builder + tick wiring (primary-only, byte-identical)

The emit side and the plumbing. Implements spec §1 and §3.

**Files:**
- Modify: `windows/vessel/src/liveness.rs` (the `alarm_field` builder; wire
  into `DriveMovements::step` and `affect_of`)
- Test: `windows/vessel/src/liveness.rs` test module (builder unit tests)

**Interfaces:**
- `pub fn alarm_field(frozen: &Ledger, npcs: &[Npc], terrain: &dyn Terrain, day: WorldTime) -> std::collections::BTreeMap<RoomAddr, f64>`
  — for each npc: compute its **primary** danger by reading `affect_of(frozen,
  npc, day, terrain)` **with the terrain's own hazards only** (no alarm field —
  this is the frozen, pre-alarm read, so emission is terrain-sourced by
  construction). If that affect is active Danger distress (`object ==
  Some(DriveKind::Danger)` and `arousal ≥ DANGER_ACT`, i.e. its felt threat
  crossed `act`), stamp the emitter's felt-threat magnitude onto its
  `agent_position` cell and each `RoomAddr::neighbors()` (`+=`, accumulating
  across emitters), then `clamp(0.0, 1.0)` every entry. Empty map when no
  creature is primary-afraid.
  - Determinism: iterate `npcs` in their given order but ACCUMULATE into a
    `BTreeMap` with `+=` (addition is commutative, so order-independent);
    clamp is applied once at the end over the sorted keys.
  - type-audit: the return is a `BTreeMap<RoomAddr, f64>` — the `f64` values are
    a bare ratio at the pub boundary → tag `bare-ok(ratio: return)`.
- `DriveMovements::step`: build the field **once** at the top of `step`
  (before the `for npc` loop) from `frozen`, `&self.npcs`, `self.terrain`, at
  `self.from`; pass `alarm: Some(&field)` into the `Danger { … }` built per
  step. (The field is fixed across the whole interval — the next-tick wave.)
- `affect_of`: for narration/health consistency the after-the-tick affect read
  should see the same alarm the tick did. Give `affect_of` an optional alarm
  argument OR a sibling `affect_of_with_alarm`; the field-BUILD path must call
  the primary-only (alarm-free) read to avoid the feedback loop. Recommend: keep
  `affect_of` primary-only (its existing signature, used by the builder) and add
  the alarm as an optional param defaulted at call sites — resolve the exact
  shape in execution, preserving: **build reads alarm-free; post-tick read sees
  the field.** Document the invariant at the builder.

**Steps (TDD):**
- [ ] **Step 1: Write failing unit tests:**
  - `alarm_field_is_empty_with_no_primary_fear`: a population on hazard-free
    terrain → `alarm_field(...)` is empty.
  - `alarm_field_haloes_a_primary_afraid_creature`: one creature on a planted
    UNCANNY-hazard cell (its Danger crosses `act`) → the field has entries at
    its cell + 3 neighbours, each `> 0` and `≤ 1`; a distant cell is absent.
  - `alarm_field_does_not_re_emit` (termination): a creature on SAFE ground
    but adjacent to a hazard cell that a FIRST creature occupies — the safe
    creature is not itself primary-afraid (its own terrain danger is below act
    once the halo is excluded), so it contributes NOTHING to the field. Proves
    borrowed alarm is not re-emitted (built alarm-free).
- [ ] **Step 2: Run — verify fail.**
- [ ] **Step 3: Implement** `alarm_field` + wire `DriveMovements::step` +

- [ ] **Step 4: Run the new tests + the full vessel suite**
