# The Type Audit — Design

**Date:** 2026-07-09
**Status:** Approved (brainstorming session)
**Campaign:** 27 (confirm numbering at merge — 24 Deep Time and 26 Live Orrery are in flight or freshly landed)
**Provenance:** Bare `f64`s crossing API boundaries have already produced real bugs (SKY-20, a sidereal-for-synodic period confusion, is the house exhibit). The typed-quantities convention exists (decision 0008, Campaign 2 design principle 5) and astronomy honors it (`units.rs`: `quantity!` newtypes with validating constructors), but nothing enforces it, and climate (~49 bare-f64 API lines), terrain (~41), kernel (~22), and settlement (~22) grew bare. This campaign audits every primitive at every public boundary, ratifies a verdict per crossing, and installs enforcement so the convention can never silently drift again. Remediation happens in later waves; this campaign changes no production code.

---

## 1. Goal and contract

Make every primitive crossing of a public API boundary a **judged, tagged, enforced** decision:

1. **The tool** — `tools/type-audit/`, a syn-based checker outside the workspace, with two commands: `check` (default-deny: any untagged pub-boundary primitive is an error) and `report` (regenerates the committed audit report).
2. **The tags** — one structured doc-comment line per judged item: `bare-ok(<class>)`, `waiver(<reason>)`, or `pending(wave-N)`. Verdicts live at the use site, travel with refactors, and are reviewed in the same diff that changes a signature.
3. **The report** — `docs/audits/type-audit-report.md`, generated by the tool and drift-checked in CI: counts by verdict class and crate, plus the pending-by-wave table that measures remediation progress.
4. **Two decision entries** — 0024 (non-workspace dev tools may use parser libraries; extends 0019's boundary) and 0025 (the bare-ok rubric, ratified from the audit's contested cases).
5. **The remediation roadmap** — waves specced in §7, executed as separate campaigns.

**Contract:** no production code changes. New files are the tool, the tags (doc-comment lines only), the report, the decisions, and CI wiring. `cargo test --workspace` output, world bytes, and all committed sim artifacts are untouched. Safe to run parallel to in-flight campaigns.

## 2. The tool

`tools/type-audit/` is a standalone Cargo project — **not a workspace member** (added to the root manifest's `exclude` list), with its own lockfile. Dependencies: `syn` (full + visit features), `proc-macro2`, `quote` if needed for spans; nothing else without cause. The workspace's serde-only allowlist and decision 0019 are untouched: the sim's build graph never sees syn. Decision 0024 codifies the carve-out: *offline dev tools outside the workspace may use parser libraries; their outputs are committed and drift-checked* — the same posture as "models author, dice roll."

**What counts as a public boundary.** An item is audited when it is syntactically `pub` (bare `pub` only — `pub(crate)`/`pub(super)`/`pub(in …)` are internal by the astronomy convention) in non-test code. Audited positions:

- `pub fn` parameters and return types (free functions, inherent methods, trait methods with default or required signatures)
- `pub struct` fields (named and tuple) that are themselves `pub`
- `pub enum` variant fields (all variant fields — variants inherit the enum's visibility)
- `pub const` / `pub static` types
- `pub type` alias right-hand sides

Excluded: `#[cfg(test)]` items, `tests/`, `examples/`, `benches/`; trait *implementations* of foreign or already-audited traits (the trait definition is the boundary, the impl repeats it); private items at any depth.

**Tracked primitives:** `f64 f32 u8 u16 u32 u64 u128 i8 i16 i32 i64 i128 usize isize bool char String &str`, including inside `Option<…>`, `Vec<…>`, slices, arrays, tuples, and references (the primitive is what's judged, wherever it nests). Generic parameters and non-primitive types are ignored.

**Syntactic honesty rule.** syn parses; it does not resolve names. A type alias that launders a primitive (`type Meters = f64;` used in a signature) is caught at the alias declaration itself (a `pub type` RHS is an audited position), not at every use — good enough, stated plainly. If a construct defeats classification, the tool errs toward reporting it (a false positive gets a tag; a silent skip is the only forbidden outcome).

## 3. The tag grammar

One line inside the item's doc comment (which `missing_docs` already guarantees exists), anywhere in the comment, at most one tag line per item:

```
/// type-audit: <verdict>[, <verdict>…]
```

where each `<verdict>` is one of:

- `bare-ok(<class>[: <position>])` — permanently fine as a bare primitive. `<class>` is a rubric class from decision 0025 (§4). The optional `<position>` names a parameter or field when one item mixes verdicts (e.g. `bare-ok(ratio: ocean_fraction)`).
- `waiver(<reason>[: <position>])` — should be a newtype; deliberately deferred with a stated reason (e.g. `waiver(decision-0014)`, `waiver(elevation-convention)`).
- `pending(wave-<N>[: <position>])` — will be converted in remediation wave N. Transitional: the wave's diff deletes the tag together with the conversion. `git grep -c 'type-audit: pending'` is the burn-down metric.

An item with multiple primitive positions may carry one tag line with comma-separated verdicts. A position-less verdict covers every audited position of the item. Unknown classes, malformed tags, tags on items with no audited primitive (stale tags), and audited primitives with no covering verdict are all `check` errors — the enforcement is bidirectional, so tags can't rot either way.

**Visibility side effect (accepted deliberately):** doc-comment tags render in rustdoc and in any book excerpt showing the item. This is judged a feature — the type decision is real documentation — and was an explicit choice, not an oversight.

## 4. The verdict rubric (becomes decision 0025)

Initial `bare-ok` classes, extended only through the audit's ratification loop (novel classes go to Nathan; the ratified set is codified in decision 0025 at campaign end):

- `ratio` — dimensionless ratio or fraction (decision 0008 / C2 design principle 5 says these *stay* bare)
- `count` — honest cardinality (cell counts, seed counts, octaves)
- `index` — position into a structure whose type carries the meaning (`CellId(pub u32)` exists precisely for the non-honest cases)
- `constructor-edge` — the sanctioned raw edges of a newtype: `new(value: f64)` params and `get() -> f64` returns
- `envelope` — the kernel `Value` enum and `Fact` fields: the trace protocol is deliberately dumb (spec §3.1.6)
- `identifier-text` — `String`/`&str` that is a name, label, or key whose contract is being plain text (stream labels, predicates — see decisions 0015, 0019)
- `render-internal` — pixel/glyph math in render modules where the quantity is screen-space, not world-space
- `flag` — a `bool` whose meaning is unambiguous at every call site (the audit judges this per-site; a `bool` param that reads as `foo(true, false)` at call sites is `pending`, not `flag`)

`waiver` reasons at minimum: `decision-0014` (`Fact.day`), `elevation-convention` (the documented bare-f64 elevation waiver from the Campaign 3 plan). Waivers cite their source; a waiver nobody can trace is a `check` error of the human kind — the reviewer's job.

Anything that is a coherent physical quantity or a confusable identity crossing a public boundary and is not in a `bare-ok` class gets `pending(wave-N)`: meters, days, Kelvin, degrees/radians, AU, masses, probabilities-used-as-thresholds, u32s that are secretly ids.

## 5. Enforcement and CI wiring

CI gains two lines in the existing "Artifacts are current" step (the drift-check family is the right home — the tool is built once, both commands run):

```
cargo run --manifest-path tools/type-audit/Cargo.toml -- check
cargo run --manifest-path tools/type-audit/Cargo.toml -- report > docs/audits/type-audit-report.md
```

followed by the existing `git diff --exit-code`, extended to cover `docs/audits/`. `check` failing fails CI (untagged primitive, stale tag, malformed tag); `report` drifting fails CI (someone changed tags without regenerating). Locally, developers run the same one-liners; they are documented in CLAUDE.md's command block. The tool is **not** part of `cargo test --workspace` — named cost, accepted: the per-commit local gate stays fast, and the drift step already polices every other committed artifact.

## 6. The audit process (how tags land during this campaign)

1. Build the tool; run `check` — the initial error list *is* the work queue (expect several hundred items).
2. Classify per crate: a pass proposes a verdict for every item — the tag line plus, for `pending` items, the wave assignment and a save-format note (does the value reach `Value`/world JSON serialization — the constraint every wave inherits).
3. Contested or novel-class items go to Nathan in one batch per crate; ratified outcomes extend the rubric.
4. Tags are committed per crate (docs-only diffs — a doc-comment line is not production code); `check` goes green crate by crate; the report and decisions 0024/0025 land last.

## 7. Remediation waves (specced here, executed as separate campaigns)

- **Wave 1 — kernel:** hoist `quantity!` from astronomy into `kernel::units` (macro_rules only, per 0019) so every domain can use it without cross-domain deps; astronomy re-exports or migrates; then the kernel's own pending rows (geosphere latitude/longitude degrees, noise coordinates, `WorldTime` consistency).
- **Wave 2 — terrain + climate:** the two largest bare surfaces. Elevation's waiver survives unless Nathan retires it here (that retirement would be a real epoch decision — it touches the documented convention, not just types).
- **Wave 3 — settlement, species, language, religion, windows.**

Every wave inherits: serialized world bytes never change (newtypes stop at `.get()` edges; `cli/tests/lens_purity.rs` and CI drift are the guards); pins and stream draws untouched; each wave's DoD includes deleting its `pending` tags and watching the report's burn-down table shrink.

## 8. Tests

The tool tests itself (its own dev-dependencies are its business, outside the workspace):

- **Extraction battery:** fixture `.rs` snippets → expected audited positions (multi-line signatures, nested generics, tuple structs, enum variants, `pub(crate)` exclusion, `#[cfg(test)]` exclusion, type-alias RHS, trait def vs. impl).
- **Tag parser battery:** every verdict form, position qualifiers, multi-verdict lines, malformed-tag rejection, stale-tag detection.
- **Bidirectional property:** for a fixture crate, `check` passes iff tags exactly cover audited positions — no more, no less.
- **Report determinism:** same tree → byte-identical report (sorted output, no timestamps).

The workspace side needs no new tests; CI wiring is exercised by the drift step itself.

## 9. Definition of Done

- Tool + tests green (its own `cargo test` in `tools/type-audit/`); `check` green across the workspace; report committed; CI step extended and passing.
- Decisions 0024 and 0025 in `docs/decisions/`; CLAUDE.md command block updated with the two one-liners; the typed-quantities paragraph gains a pointer to the tag grammar.
- Chronicle entry; freshness sweep; retrospective (decision 0020).

## 10. Non-goals

- **No newtype conversions** — not one signature changes in this campaign; that is Waves 1–3.
- **No workspace dependency changes** — the allowlist is untouched; the tool lives outside.
- **No relitigation of ratified bare choices** — 0014 and the elevation convention become waiver tags citing their sources, full stop.
- **No custom attribute machinery** — tags are doc-comment lines because syn sees them as `#[doc]` and `missing_docs` already guarantees the comment exists; stable Rust offers no inert custom attributes without proc macros (0019).
