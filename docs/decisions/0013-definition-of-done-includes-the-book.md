# 0013. Definition of Done includes the project book

**Status:** Accepted (2026-07-05) · **Decider:** Nathan

In the context of a solo project with a history of resets after long
unobservable stretches, facing the risk that the published book drifts behind
merged reality, we decided that **the Definition of Done for every merged plan
includes the project book: a chronicle entry plus a freshness sweep of stale
chapters**, accepting that no campaign is complete until the book reflects it.

**Context.** The book is the comprehension mechanism — the gate is that Nathan
can explain a campaign from the book alone. A book that lags reality fails that
gate and re-creates the unobservable-progress spiral that has ended prior
projects. Book prose is written at a deliberate technical/mathematical
altitude, comprehensible without the code.

**Consequence.** Every campaign closes with a chronicle entry and a stale-chapter
sweep; generated artifacts (almanacs, registry, manifests, Lab summaries) are
drift-checked in CI so the book's *data* can never silently rot. Analysis prose
is gated by Nathan's comprehension review.
