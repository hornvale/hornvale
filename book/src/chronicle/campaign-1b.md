# Campaign 1b: The Tier-0 Cascade

**July 2026 · 15 commits · outcome: complete, merged**

## What was attempted

Give the kernel its first real consumers: the simplest possible implementation
of *every* domain, all at once — astronomy, climate, terrain, settlement,
culture, religion — plus the first two windows (the almanac and the REPL) and
a `hornvale` command-line tool to hold them. The exit criterion, verbatim from
the spec: create a world from seed 42, interrogate its sky, village, and
beliefs interactively, and render a one-page world document — all
deterministically.

## What landed

All of it, proven end-to-end: a test suite drives the actual built binary
through the exit criterion, including byte-identical almanacs across separate
processes. The world of seed 42 now contains the goblin village of
**Gruugish**, population 57, whose people revere the sun that never sets.

The structural claims of the constitution stopped being aspirations this
campaign:

- **Domains depend only on the kernel** — verified three ways (manifests,
  dependency lock, source scan). No domain can see another.
- **The trace protocol works.** Religion's tests feed it a *periodic*
  phenomenon — a pale wanderer with a thirty-day cycle — which no tier-0
  provider can produce. Religion mythologizes it correctly anyway, proving it
  genuinely neither knows nor cares what is upstream. When real moons arrive
  in Campaign 2, religion is already ready for them.
- **Refinement got its first real client.** Gruugish's name was not merely
  picked; it was chosen through the consistency engine, which will refuse any
  name that contradicts a committed fact. With an empty world this constraint
  is trivially satisfied — but the pathway is now the one every future naming
  decision flows through.

## What was learned

- **A tale of two villages.** The First Light artifact calls seed 42's
  village *Bolnar*; the real cascade calls it *Gruugish*. Same seed, same
  world — different generation code drawing from different candidate lists.
  This is determinism working exactly as designed, and a vivid lesson in what
  "the same seed" actually promises: the same world *under the same
  generation contracts*. The labels of the derivation chains and the
  candidate pools they draw from are save-format commitments, as permanent as
  the hash constants.
- **Generated documents need assertions only the feature can satisfy.** A
  review caught that the almanac silently ignored the phenomena handed to it
  — and the test meant to catch this passed anyway, because the string it
  looked for happened to appear elsewhere in the document. The fix was to
  render the phenomena and assert on text nothing else produces. Lesson
  absorbed into the testing standard: a coverage assertion that *can* be
  satisfied by coincidence eventually *will* be.
- **The anti-drift loop must regenerate through the same path reality uses.**
  The `concepts` command initially rebuilt its registry by hand rather than
  from a built world — meaning CI's drift check compared the copy against
  itself and could never catch the kernel's real registration drifting. The
  final review caught it; the dump now comes from an actual world. A
  self-consistent lie is the failure mode of every generated-documentation
  scheme, and the guard is always the same: generate from the artifact, never
  from a parallel reconstruction.

## Deferred, deliberately (Campaign 2's opening chores)

Centralizing provider construction (three call sites currently know how to
build the sun — one-site replacement is a precondition for real astronomy);
a small kernel-ergonomics batch (a shared name-predicate constant, a text
lookup accessor, standard derives on the report types); extracting a library
target from the CLI so the planned in-browser demo can reuse the REPL, whose
buffer-generic design is already WASM-shaped.
